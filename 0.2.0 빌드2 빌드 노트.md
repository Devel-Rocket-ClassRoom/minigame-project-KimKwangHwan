## 1. 이번 빌드에 새로 포함된 / 확장된 기능

### 1.1 플레이어
- **체력 / 스태미너 시스템**
  - `PlayerHealth` — 데미지/사망 이벤트(`OnDamaged`, `OnDead`), 최대 HP 관리
  - `PlayerStamina` — `TryUseStamina(amount)`, 사용 후 회복 시작 딜레이(`genStartInterval`) + 주기적 회복(`genInterval`, `genAmount`)
  - `UI/HpBar`, `UI/StaminaBar` — 이벤트 구독 기반 실시간 게이지
- **사망 / 피격 상태**
  - `PlayerDeathState`, `PlayerHurtState` 추가
- **콤보 공격 확장**
  - 지상 콤보 공격 2개 추가 (`PlayerAttackState` + `AttackRuntime` 정리)
- **이동 / 점프 다듬기**
  - 더블점프·벽점프 보정
  - **아래 점프(드롭다운)** 구현 (`PlayerMotor` 보강, `PlayerInputReader` 입력 추가)

### 1.2 적
- **공통 적 시스템 보강**
  - `EnemyHealth` 신설 — 적측 HP/데미지 파이프라인 통일
  - `EnemyAnimEvent` — `HitboxOn`/`HitboxOff`/`ProjectileFire`/`RecoveryEnd`/`TelegraphEnd` 이벤트 브로드캐스트
  - `EnemyContext.WaitForAnimEvent` / `ArmEvent` 도입 → 패턴 코루틴이 애니 이벤트와 정확히 동기
  - `EnemyCombat` 베이스로 `SelectPattern` / `MarkPatternUsed` 끌어올려 중복 제거
  - `EnemyMotor`에 `SuspendControl` / `ResumeControl` 추가 (점프 공격 중 외부 속도 제어)
- **마법사 해골(Mage Skeleton) 확장**
  - `MageSkeletonHurtState`, `MageSkeletonDeathState` 추가
  - `ProjectileAttack`이 애니 이벤트(`ProjectileFire`/`RecoveryEnd`) 기반 발사·복귀
- **근접 적: 어쌔신(Assassin) 신규**
  - `Assassin.cs` + `AssassinIdle/Move/Attack/Hurt/Death` 상태 머신
  - `AssassinCombat`
  - 패턴 SO: `AssassinMeleeAttack`, `AssassinJumpMeleeAttack`(점프 근접)
- **돌진형 적: ElectricGolem 신규(미완)**
  - `ElectricGolem.cs` + `Idle/Move/Attack/Hurt/Death` 상태 머신
  - `ElectricGolemCombat`
  - 패턴 SO: `ElectricGolemMeleeAttack`

### 1.3 보스(Witch) — 신규
- **컨트롤러**: `Witch.cs` (`EnemyController` 상속) + `WitchIdleState` / `WitchMoveState` / `WitchAttackState`
- **컨텍스트 / 선택기**: `BossContext`, `PatternSelector` (쿨다운 + 가중치)
- **보스룸**: `BossRoom` — 영역 정의 + `RandomPoint`, `RandomPointAwayFrom`
- **이동 행동**: `BossMoveBehavior`, `TeleportMove` — 패턴별 위치 이동 전략
- **애니 이벤트**: `BossAnimEvent`
- **공격 패턴 SO 7종 (+ 보조 1)**
  - `ComboAttackPattern`, `CometDivePattern`, `DashPattern`, `FireBallPattern`,
    `HardCastPattern`, `InfernoPattern`, `ThunderBoltPattern`, `ThunderTwinPattern`

### 1.4 공통 시스템
- **오브젝트 풀** — `ObjectPool`, `PoolManager`, `PooledObject` 신규 (투사체·이펙트 재사용)
- **투사체 / 이펙트 확장**
  - `HomingProjectile` (유도 투사체)
  - `Stationary` (장판)
  - 기존 `Projectile` 풀링/소멸 처리 정비
- **데미지 파이프라인** — `HurtBox` 분리, 적/플레이어 양측에서 동일 인터페이스 사용
- **카메라** — Cinemachine 카메라로 교체

## 2. 빌드 환경

- **Unity**: 6000.3.15f1
- **렌더링**: Universal Render Pipeline (URP)
- **그래픽스 API**: Direct3D 12
- **입력**: Unity Input System (`PlayerInputAction.inputactions`)
- **물리**: 2D Physics + `PlayerPhysicsMaterial`
- **카메라**: Cinemachine

---

## 3. 빌드 절차

1. Unity Editor → `File > Build Settings`
2. 플랫폼: PC, Mac & Linux Standalone (Windows / x86_64)
3. 출력 경로: `./Build/`
4. 산출물 확인: `Build/TreasureHunter.exe` 실행
5. (배포용) `Build.zip` 으로 압축

---

## 4. 알려진 제한사항 / 이슈

- `BossRoom` 내에 `Player` 인식 시, `Witch`가 `Player`를 공격하도록 수정해야 함
- 일반 몹 벽이나 낭떠러지 인식 시, 반대로 이동하도록 수정해야 함
