## 3. 잘된 점

1. **보스 패턴 SO화** — `BossPattern` 추상 클래스 + `PatternSelector` 조합으로 8종 패턴을 데이터 자산만으로 추가/조정 가능.
2. **애니메이션 이벤트 동기화** — `WaitForAnimEvent` / `ArmEvent`로 패턴 코루틴이 애니 타이밍과 정확히 맞물려 히트박스/투사체 발사 누락 가능성 감소.
3. **공통 베이스 일원화** — `EnemyCombat`로 `SelectPattern` 등을 끌어올려 신규 적 추가 비용을 더 낮춤 (Assassin/ElectricGolem 둘 다 베이스 클래스만 재사용).
4. **오브젝트 풀 도입** — 다발성 투사체·이펙트의 GC/스파이크 위험 완화.

## 4. 이슈 / 개선 필요 사항

- `EnemyCombat`의 `FindWithTag("Player")` 자체를 안전한 참조 방식으로 교체 필요 (주석에도 표기됨).
- 게임 오버 / 리스폰 / 메뉴 화면 미구현 — 게임 루프 미완.
- 스테이지가 여전히 테스트용 1개. 본격 레벨 디자인 필요.
- 벽타기 중 방향키를 눌렀을 때 떨어지는 현상 (빌드1 잔존) 재확인 필요.
- `BossRoom` 내에 플레이어를 인식했을 때, 보스가 플레이어 공격
- 일반 몹 벽, 낭떠러지 인식 시, 반대로 방향을 트는 기능 필요
