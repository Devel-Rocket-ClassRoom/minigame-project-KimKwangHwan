## 1. 이번 빌드에 포함된 기능

### 1.1 플레이어
- **상태 머신 구조**(`PlayerStateMachine` + `PlayerState` 서브클래스)
  - `Idle`, `Run`, `Air`, `Jump`, `Fall`, `WallClimb`, `Attack`, `Dash` 상태 분리
- **이동/점프**
  - 좌우 이동, 더블 점프, 점프 후 즉시 벽잡기 가능
- **벽타기 / 벽점프**
  - 벽 방향에 따른 캐릭터 좌우 반전(`WallFlip`)
  - 벽타기 애니메이션 적용
- **공격**
  - 지상 / 공중 근접 공격 (`PlayerAttackState`)
  - 지상 / 공중 원거리 공격 (총알 → 파티클 이펙트 적용)
  - 공격 히트박스(`HitBox`) 세팅 완료
- **대쉬**
  - 지상/공중 대쉬 (`PlayerDashState`)
  - 쿨다운(`dashCooldown`), 지속시간(`dashDuration`), 잔상(`AfterImage`) 효과
  - `dashSpeed = 20`, `dashDuration = 0.7s`, `dashCooldown = 0.5s` (기본값)
- **피격**
  - `PlayerHealth.OnDamaged` → `Hurt` 애니메이션 트리거

### 1.2 적 (마법사 해골 / Mage Skeleton)
- **공통 적 컨트롤러**(`EnemyController`) + **상태 머신**(`EnemyStateMachine`)
  - `MageSkeletonIdleState`, `MageSkeletonMoveState`, `MageSkeletonAttackState`
- **인지(Perception)**: `EnemyPerception`이 타겟(플레이어)을 감지/갱신
- **이동 AI**: 타겟 추적 이동
- **피격 AI**: 피격 시 타겟 재설정 (`perception.SetTarget()`)
- **공격 패턴(ScriptableObject 기반)**
  - `MeleeAttack` — 근접 히트박스 공격 (windup → active → recovery)
  - `DashAttack` — 텔레그래프 후 돌진 박치기
  - `ProjectileAttack` — 다발 투사체 발사 (확산각, 발사 간격, 조준 옵션)
- **투사체(`Projectile`)** 와 발사 위치(`muzzle`) 연동

### 1.3 공통 시스템
- **데미지 파이프라인**: `IDamageable`, `DamageInfo`, `HitBox`, `HurtBox`
- **잔상(AfterImage)** 컴포넌트 (대쉬용)
- **ScriptableObject 기반 공격 데이터**
  - `PlayerData`, `EnemyData`, `EnemyAttackPattern`(추상) + `Melee/Dash/Projectile`
- **Input System**: `PlayerInputAction.inputactions` 기반 입력

## 2. 빌드 환경

- **Unity**: `ProjectSettings/ProjectVersion.txt` 참조
- **렌더링**: Universal Render Pipeline (URP)
- **그래픽스 API**: Direct3D 12
- **입력**: Unity Input System (`PlayerInputAction.inputactions`)
- **물리**: 2D Physics + `PlayerPhysicsMaterial`

---

## 3. 빌드 절차

1. `main` 브랜치에 `features/10-damage` 머지 (혹은 현재 브랜치에서 바로 빌드)
2. Unity Editor → `File > Build Settings`
3. 플랫폼: PC, Mac & Linux Standalone (Windows / x86_64)
4. 출력 경로: `./Build/`
5. 산출물 확인: `Build/TreasureHunter.exe` 실행
6. (배포용) `Build.zip` 으로 압축

---

## 4. 알려진 제한사항

- 적 종류는 마법사 해골(MageSkeleton) **1종**만 구현됨
- `ProjectileAttack.Execute` 내 확산 각도 로직이 `Random.Range(-baseAngle, baseAngle)` 로 임시 처리되어 있어, 의도된 확산 패턴(`startAngle + step * i`)이 주석 처리됨 → 다음 빌드에서 정리 필요
- 데미지/피격 처리는 들어가 있으나 **HP UI, 게임 오버, 리스폰** 등 게임 루프는 없음
- 스테이지/씬은 테스트용 1개 수준
