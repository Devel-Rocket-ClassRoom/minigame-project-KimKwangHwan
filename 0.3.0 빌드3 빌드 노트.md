## 1. 이번 빌드에 새로 포함된 / 확장된 기능

### 1.1 맵 시스템 (신규)

- **`MapManager`** — Additive 씬 로드/언로드 관리 싱글톤
- **`MapData`(ScriptableObject)** — 맵 ID, 씬 이름, 연결 맵 정보 데이터 자산
- **`MapTransitionZone`** — 맵 경계 트리거, 인접 맵 로드 요청 및 플레이어 위치 이전
- **씬 구성**: `Map1` ~ `Map9` 총 9개 맵 배치 완료 (보스룸은 `Map9`)

### 1.2 세이브 시스템 (신규)

- **`SaveManager`** — 3슬롯 세이브/로드, JSON 직렬화 + `CryptoHelper` 암호화
- **`SaveDataV1`** — 저장 데이터 스키마: `savePointId`, `mapId`, `activatedSwitchIds[]`, `timestamp`
- **`SavePoint`** — 플레이어 진입 시 현재 위치·상태 저장, 재진입 스폰 기준점
- 세이브 포인트 활성화 → 스위치 상태 포함 전체 게임 상태 영속화

### 1.3 스위치 / 문 (신규)

- **`Switch`** — 플레이어 상호작용으로 트리거; 스프라이트 비활성/활성 2단계, 세이브 데이터와 연동하여 재로드 시 상태 복원
- **`Door`** — 부드러운 열림·닫힘 애니메이션, `Switch` 이벤트 수신 → 즉시 재정의(override) 가능

### 1.4 함정 시스템 (신규)

- **`Trap`(추상)** / **`Spear`** — 환경 해저드, 충돌 시 `IDamageable` 파이프라인 통해 플레이어 데미지

### 1.5 신규 적: 항아리 생물 (PotCreature)

- `PotCreature.cs` + `PotCreatureIdle/Move/Attack/Hurt/Death` 상태 머신
- 기본 근접 패턴 1종, 기존 `EnemyCombat` 베이스 재사용

### 1.6 보스룸 확장

- **`BossRoom`** 입장 트리거 — 플레이어 감지 시 보스 전투 시작(Witch 활성화), 입구 문 제어
- Map9에 보스룸 배치, MapManager와 Additive 로드 연동

## 3. 빌드 절차

1. Unity Editor → `File > Build Settings`
2. 플랫폼: PC, Mac & Linux Standalone (Windows / x86_64)
3. 출력 경로: `./Build/`
4. 산출물 확인: `Build/TreasureHunter.exe` 실행
5. (배포용) `Build.zip` 으로 압축
