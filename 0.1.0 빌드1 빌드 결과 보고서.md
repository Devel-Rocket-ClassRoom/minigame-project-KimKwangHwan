## 3. 잘된 점

1. **상태 머신 분리** — 플레이어/적 모두 `State` 단위로 코드를 잘게 쪼개 두어, 신규 액션/패턴 추가 비용이 낮다.
2. **ScriptableObject 기반 공격 데이터** — `EnemyAttackPattern` 추상 클래스를 두고 `Melee/Dash/Projectile` 로 파생시켜, 디자이너 친화적으로 값 조정이 가능하다.
3. **공통 데미지 인터페이스(`IDamageable`)** — 플레이어/적이 동일한 인터페이스를 통해 데미지를 받으므로 확장에 유리.
4. **시각 효과** — 대쉬 잔상(`AfterImage`), 총알 → 파티클 변환 등으로 액션감을 보강.
